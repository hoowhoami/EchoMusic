## [2.2.2-beta.1] - 2026-04-27

### 重构

- 播放引擎从浏览器内置音频替换为 mpv，独立进程播放，播放器崩溃不影响界面
- MKV 人声/伴奏音轨提取改为 mpv 原生处理，切换更快更稳定
- 音量均衡和淡入淡出适配新引擎
- 音频输出设备改为从 mpv 获取，设备列表更准确，切换更可靠

### 新增

- 新增「独占音频设备」开关，绕过系统混音器直接输出，获得更高音质
- 新增「自动检查更新」开关，可在设置中关闭启动时自动检查更新
- Loading 页面增加播放引擎就绪检查，引擎异常时阻止进入主界面并支持重试
- 个人中心退出登录增加确认弹窗，防止误触

### 优化

- 支持更多音频格式
- 音频解码质量更高，mpv 使用专业级 FFmpeg 解码器
- 播放更稳定，不再出现浏览器偶发的卡顿、静音、加载失败等问题
- 拖动进度条定位更精准
- 网络音频缓冲更智能，弱网环境下播放更流畅
- 切换音质/音效时保持播放进度，体验更连贯
- mpv 初始化移至窗口创建前，渲染进程启动时引擎已就绪
- 歌词页面非写真模式亮色主题下背景遮罩改为深色半透明，改善白色歌词可读性
- 歌词页面播放按钮和写真模式封面卡片统一为毛玻璃风格
- 音频设备状态提示从独立提示条改为输出设备描述小字，布局更紧凑

### 修复

- 修复打包后 mpv 二进制路径解析失败的问题
- 修复 Windows 上 mpv 进程启动慢导致 IPC 连接超时的问题
- 修复 macOS 打包后 dylib 依赖加载失败的问题（SIP 剥离 DYLD 环境变量）
- 修复 mpv 进程被信号杀死后无限重启的问题
- 弹窗（Dialog）z-index 低于播放栏，导致更新弹窗等被播放栏遮挡

### 已知变更

- 应用包体积略有增加

## [2.2.1] - 2026-04-24

### 新增

- 音效新增「人声」和「伴奏」选项，支持从 MKV 双音轨中分离播放
- 基于 Electron 自定义协议（mkv-extract://）实现音轨提取，纯 JS MKV/EBML 解析器，零外部依赖
- 已提取的音轨数据内存缓存，同一首歌切换人声/伴奏秒切，支持 Range 请求和精确 Content-Length
- 写真模式歌词自动收起：无操作后自动收起歌词至底部两行并隐藏所有按钮，支持开关和延迟配置（5-60s）
- 写真模式按钮自适应颜色：根据写真背景亮度自动切换按钮和控件的深浅配色（Canvas 采样按钮位置像素亮度），支持开关
- 歌词页面改为覆盖层模式，解决打开/关闭歌词导致 KeepAlive 缓存失效的问题
- 歌词页面打开/关闭过渡动画
- 字体设置功能

### 优化

- 写真模式下深色/浅色主题统一表现，按钮风格不再受系统主题影响
- 写真模式封面卡片宽度收窄，控制器上边距优化
- 歌词页面弹出层（背景设置、字体、翻译等）z-index 适配覆盖层模式
- 播放队列改为直接拖拽排序，移除排序模式按钮
- 桌面歌词锁定/解锁交互优化：改用 mousemove 实时检测鼠标位置控制穿透，锁定后无需移出窗口即可再次操作
- 桌面歌词文字阴影改为轻柔投影风格
- 桌面歌词支持多显示器：记住窗口所在显示器位置，下次打开时恢复到原显示器
- 实验性功能新增「禁用 GPU 加速」开关，遇到界面花屏等渲染异常时可尝试开启（重启生效）

### 修复

- 修复快速切歌时歌词可能一直显示「加载中」的问题（竞态条件）
- 修复退出登录后换账号登录，个人信息仍显示上一个账号的问题（退出时清空 API 缓存）
- 修复歌词页面快捷键无法打开的问题
- 修复打开/关闭歌词页面导致 KeepAlive 缓存失效的问题
- 修复「我最喜爱」页面定位当前歌曲位置偏移的问题（吸顶元素 class 未被识别）
- 修复 Windows 下桌面歌词每次打开位置偏移的问题（DPI 缩放坐标舍入累积）

- 切换音质或音效时保留进度条状态，消除 duration 重置导致的进度条闪烁
- 音量均衡在接口未返回有效响度数据时不再错误应用默认衰减
- 桌面歌词行切换改为窗口端本地计算，消除 IPC 同步延迟导致的跳转滞后
- 桌面歌词单行模式预渲染下一句（视觉隐藏），切换时走 move 动画，尽量消除快歌切换延迟
- KeepAlive 页面保留滚动位置，返回时不再重置到顶部
- 统一页面歌词已播/未播颜色默认值，消除首次安装时各处 fallback 不一致的问题
- 歌手详情页展示粉丝数、生日、MV 数量，关注状态改为从用户关注列表查询
- 专辑详情页展示专辑类型和语种，吸顶栏增加收藏按钮

- 侧边栏布局间距调整，头像区域上移，歌单分类与导航间距收紧

## [2.2.0] - 2026-04-21

### 新增

- 搜索页新增「歌词」和「MV」搜索标签页，支持按歌词内容和 MV 搜索
- MV 搜索结果以卡片网格展示，复用已有 MV 卡片组件
- 标题栏搜索框展开无内容时，右侧显示「搜索页」快捷入口按钮
- 新增音量均衡功能，基于曲目 LUFS 响度数据自动调整增益，使不同歌曲播放音量保持一致

### 优化

- 播放引擎从 Howler.js 重构为原生 HTML5 Audio，消除中间层导致的事件丢失和状态不同步问题
- 通过 Web Audio API（GainNode）实现音量均衡，复用同一音频元素，切歌无需重建处理链路
- 淡入淡出重构为双 GainNode 架构（fadeGainNode + normGainNode），淡入淡出使用 Web Audio API 原生 linearRampToValueAtTime 调度，不依赖 JS 定时器，窗口最小化时也能正常工作
- 歌词页面逐字动画升级为渐变填充效果，与桌面歌词一致，基于 rAF 实时插值实现逐字平滑过渡
- 直接监听原生 audio 的 durationchange 事件，修复流式音频 duration 不准确的问题

### 修复

- 修复播放中突然卡住、进度条停止且拖动无响应的问题
- 修复歌曲进度条已到末尾但 ended 事件未触发、延迟切换下一首的问题
- 修复自动切歌时淡入淡出导致歌曲没声音的问题
- 修复英文歌词中 HTML 实体未解码的问题（如 `&apos;` 显示为原始文本）
