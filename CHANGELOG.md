## [2.2.3-beta.4] - 2026-04-29

### 新增

- 新增「听歌识曲」功能，支持麦克风录音和系统音频捕获两种识别方式
- 识别成功后展示歌曲信息，支持一键播放、收藏、添加到歌单、查看详情
- TitleBar 新增听歌识曲按钮
- Windows/Linux 标题栏新增全屏按钮，支持切换全屏模式

### 修复

- 修复专辑详情页超过 30 首歌曲时无法加载全部歌曲的问题
- 修复专辑中同一首歌的不同版本因 id 冲突导致列表重复渲染的问题

### 重构

- 播放引擎从 mpv 子进程迁移到 libmpv 内嵌方案，通过 Rust NAPI addon 直接调用 libmpv C API
- 淡入淡出从 TypeScript setInterval 轮询改为 Rust 线程高精度定时，不受 JS 事件循环影响
- 渲染进程 mpv preload API 补充完整 TypeScript 类型声明，消除 any 类型

### 优化

- 大歌单加载优化：首页数据秒出即渲染，剩余页后台并发加载，大幅缩短总加载时间
- 播放全部优化：无需等待歌单全部加载完成，先播已加载歌曲，后台自动补全播放队列
- 收藏歌曲、歌单详情、歌手详情、专辑详情统一使用通用分页加载器，减少重复代码
- 优化歌词页面打开动画流畅度，减少 GPU 渲染压力
- 降低播放时渲染进程 CPU 占用，歌词相关计算改为按需执行
- 播放器底栏歌曲信息滚动动画不再无限循环，降低待机功耗
- 暂停/播放操作响应更快
- 切歌时音量过渡更平滑，通信开销更低
- 播放命令从 JSON IPC socket 序列化改为直接函数调用，通信延迟降至零
- 音量合成器中天然显示为 EchoMusic，无需 --audio-client-name 伪装
- 不再需要 spawn/kill/重启等进程管理逻辑
- 当播放时长与原曲差异较大时，在页面歌词和桌面歌词中提示歌词可能不同步

### 修复

- 修复单曲循环播放结束后卡住不重播的问题
- 修复切换音效后进度条可能越界导致播放卡住的问题
- 修复拖动进度条可能越界的问题

## [2.2.0] - 2026-04-21

### 新增

- 搜索页新增「歌词」和「MV」搜索标签页，支持按歌词内容和 MV 搜索
- MV 搜索结果以卡片网格展示，复用已有 MV 卡片组件
- 标题栏搜索框展开无内容时，右侧显示「搜索页」快捷入口按钮
- 新增音量均衡功能，基于曲目 LUFS 响度数据自动调整增益，使不同歌曲播放音量保持一致

### 优化

- 播放引擎从 Howler.js 重构为原生 HTML5 Audio，消除中间层导致的事件丢失和状态不同步问题
- 通过 Web Audio API（GainNode）实现音量均衡，复用同一音频元素，切歌无需重建处理链路
- 淡入淡出重构为双 GainNode 架构（fadeGainNode + normGainNode），淡入淡出使用 Web Audio API 原生 linearRampToValueAtTime 调度，不依赖 JS 定时器，窗口最小化时也能正常工作
- 歌词页面逐字动画升级为渐变填充效果，与桌面歌词一致，基于 rAF 实时插值实现逐字平滑过渡
- 直接监听原生 audio 的 durationchange 事件，修复流式音频 duration 不准确的问题

### 修复

- 修复播放中突然卡住、进度条停止且拖动无响应的问题
- 修复歌曲进度条已到末尾但 ended 事件未触发、延迟切换下一首的问题
- 修复自动切歌时淡入淡出导致歌曲没声音的问题
- 修复英文歌词中 HTML 实体未解码的问题（如 `&apos;` 显示为原始文本）
